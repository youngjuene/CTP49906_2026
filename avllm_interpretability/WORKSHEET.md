# Lab worksheet — hypothesis before ▶

**Used in W9-W11.** One row per run. Fill the *Hypothesis* column **before** you press ▶ —
that's the whole exercise. A refuted hypothesis and an uninformative run are results too;
record them. This is the same question format your W11 project is graded on, so by then
you'll have practiced it a dozen times.

**The habit:** *"If I block X, the result should change like Y."* Then check whether the
model agrees — and pair anything interesting with a control that can fail
(e.g. the silent-track clip `assets/02321_silent.mp4`).

**For every effect you plan to keep, answer both halves (this is the W11 rubric question):**

> My control is: ___________________________
> If my control **passes**, what is true? ___________________________
> If my control **fails**, what is true? ___________________________
> *(If the honest answer to either is "nothing", it isn't a control yet.)*

# 실험 워크시트 — ▶를 누르기 전에 가설부터

**W9-W11에서 사용.** 실행 하나당 한 줄. *가설* 칸은 ▶를 누르기 **전에** 채우세요 —
그것이 이 연습의 전부입니다. 반증된 가설도, 아무것도 보여 주지 않은 실행도 결과입니다;
기록하세요. W11 프로젝트가 채점되는 질문 형식과 같으므로, 그때쯤이면 이미 열 번 넘게
연습한 상태가 됩니다.

**습관:** *"X를 막으면 결과가 Y처럼 바뀔 것이다."* 그런 다음 모델이 동의하는지 확인하고 —
흥미로운 것은 무엇이든 실패할 수 있는 대조군과 짝지으세요
(예: 무음 트랙 클립 `assets/02321_silent.mp4`).

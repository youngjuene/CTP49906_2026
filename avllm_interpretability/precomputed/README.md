# Precomputed artifacts (F5a — GPU-free replay)

When `USE_PRECOMPUTED = True` in `CTP49906_avllm_molab.py`, the notebook replays
the fixed-parameter **W7-W9** plots from the files in this directory instead of
running the model — so the class survives a molab GPU outage (`USE_PRECOMPUTED`
defaults to `False`; this is a break-glass mode).

## What lives here

Generated by `scripts/generate_precompute.py`, for the sample clip
`assets/02321.mp4`:

| File | Feeds |
|---|---|
| `logit_lens_audio_token_analysis.csv` | W8 logit-lens diversity / dominance plots |
| `captions.json` | the generated caption + baseline/knockout captions |
| `attention_summary.json` | reduced baseline + knockout generated-query attention-mass matrices (decode steps only; and delta in the notebook) |
| `meta.json` | clip / prompt / layer-count provenance |

Raw attention tensors are **not** committed (one eager prefill layer is ~64 MB
fp32) — only the reduced matrices. Schema v2 stores both conditions while keeping
the legacy top-level knockout matrix fields readable by older notebooks.
Replay validates `meta.json` against the notebook's fixed clip, prompts, frame
count, generation length, capture layers, knockout rules, model, and immutable
model revision; a mismatch fails loudly instead of relabeling stale results.

## Regenerating

Run once on a GPU (molab or a CUDA box), from the repo root:

```bash
python avllm_interpretability/scripts/generate_precompute.py
```

Re-run whenever the sample clip, prompts, or model change. The interactive
playground and teacher-forcing sections are **not** precomputable (arbitrary
student inputs); they still require a GPU and fail loudly if submitted while
`USE_PRECOMPUTED = True`.

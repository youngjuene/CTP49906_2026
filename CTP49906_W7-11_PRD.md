## 3. Build specs

### F1 — Teacher forcing (`answer` token type) 🔴 Top priority

**Problem**
The playground runs a single forward pass, so `generated` tokens don't exist. But `generated → audio`
is the first thing a student tries — it's the most intuitive rule. Nothing happens (the notebook and
README already have to *warn* about it). They lose interest. F1 turns that documented dead-end into a
graded, continuous, deterministic measurement.

**Feasibility (verified).** The hard parts already exist, so v1's "riskiest code" framing overstates
the difficulty — worth knowing for the Aug-31 scheduling. `BlockAttentionHook`'s **prefill branch**
fires whenever `q_len == k_len == original_input_len` (`src/attention_knockout_experiment.py:117-144`),
so forwarding `[prompt, C]` at the extended length exercises exactly the mask code path the playground
already drives daily. Appending is a `torch.cat` on `input_ids` + `attention_mask` only — the
audio/video feature tensors are untouched (placeholder scatter matches by token id; the trailing text
gets the same teacher-forced RoPE positions it had during generation). Per-position
`log P(C_t | C_<t)` is one `log_softmax` over the forward pass's logits, shifted by one.

**Solution**

1. **In the playground run cell** (using the already-loaded **eager** `attention_model`), generate the
   baseline caption `C` with **greedy decoding** (`do_sample=False`), **cached** keyed on
   `(clip hash, prompt, nframes)` so unchanged submits skip regeneration. Slice the **raw generated
   ids**: `C_ids = gen[:, input_len:]`. **Never re-tokenize the decoded string** — `batch_decode(...,
   skip_special_tokens=True)` drops the audio/video placeholders and does not round-trip, which would
   break feature scatter. *(v1 said "generate C once in the slow cell"; that caption is bound to the
   params-cell clip/prompt and would be teacher-forced onto whatever the student uploaded — a
   mismatched-audio measurement mislabeled as self-belief Δ, and it breaks W11's student clips.)*
2. Append `C_ids`; `attention_mask = cat([mask, ones])`. Forward pass over `[prompt, C]`.
3. Tag `C`'s positions as **`answer`** — **positionally** (`prompt_types + ["answer"] * len(C_ids)`),
   **not** by re-running a token-id mapper, which would type them `query_text`. Add `"answer": 5` to
   `TOKEN_TYPE_MAP`.
4. Pass `original_input_len =` the **full extended length**, so the prefill branch triggers
   (`q_len == k_len == extended_len`). With prompt-only length neither hook branch fires and the
   knockout silently no-ops.
5. Apply knockout `answer → audio`, layers `[a, b)`. Run a **knockout pass and a baseline pass** (same
   eager model, same encoded inputs, both greedy).
6. Read logits at each `C` position; compute `Σ_t log P(C_t | C_<t, prompt)` for knockout and baseline.
   Metric: **Δ log-likelihood, `Δ = knockout − baseline`** — how much less the model believes what it
   said. *(Sign convention pinned to match the existing "Δ diversity (knockout − baseline)" panel.)*

**Guardrail (fail loud, not silent).** Convert `block_attention`'s `KeyError` handler
(`src/attention_knockout_experiment.py:184-196`) from **print-and-yield-with-zero-hooks** into a
`raise`. As written, a mistyped or unknown token type runs a **baseline labeled as a knockout** — the
exact plausible-but-wrong artifact the course teaches students to distrust. The playground's
`except Exception` handler already surfaces a raise as an in-notebook callout. Teacher-forced types
feeding `block_attention` must come from the attention mapper (positional `answer`), never the
logit-lens id mapper.

**Acceptance criteria**

- [ ] `answer` appears in `TOKEN_TYPE_MAP` and prints in `token counts:`; dropdown shows `answer`.
- [ ] **Student-facing surface updated for the new source** (parallel to F4's README-revision
  deliverable). Teacher forcing is its **own** playground section (source fixed to `answer`, the student
  picks the target and layer band) — deliberately **not** a new option in the *diversity-scoreboard*
  dropdown (`CTP49906_avllm_molab.py:607,711`), where `answer` would be inert exactly as `generated` is
  (that scoreboard runs a prompt-only forward pass with no answer tokens). Add `answer` to the README
  token-type table and revise the "Why `generated` does nothing" callout (`README.md:113-141`) to point
  at teacher forcing as where an `answer` source becomes meaningful.
- [ ] `answer → audio` produces an aggregate Δ **below a stated negative margin** (not merely "non-zero"
  — float noise passes non-zero once any key is masked) on a clip with an audio-dependent caption.
- [ ] **Sanity check, made falsifiable:** on a committed **silent-track** asset (audio stream *present*,
  all-zeros — so real audio tokens exist to be blocked), `|Δ_silent| / token ≪ |Δ_audio-dependent| /
  token` on the same frames, prompt, and layers, within a stated tolerance. *(A clip with **no** audio
  track produces zero audio tokens, so `answer → audio` blocks nothing and Δ = 0 vacuously — it passes
  even if teacher forcing is completely broken. A track-less clip must instead raise the existing
  no-audio-tokens warning; that is not a substitute for this check.)*
- [ ] **Deterministic:** `C` decoded greedily with `do_sample=False` pinned **explicitly** — the shipped
  `generation_config.json` is an empty stub that resolves to greedy today, so this is spec-hardening
  against a silent upstream flip (the weekly 8 GB re-download, R5, would deliver one straight to class),
  not a live bug fix. Test: run the same submit twice, assert **bitwise-equal per-token Δ vectors**
  (same model instance, no re-encode).
- [ ] **Per-submit budget** (not "one forward pass" — Δ needs knockout + baseline = two): a warm-cache
  submit completes in **≤ 2 forward passes / ≤ N seconds** on molab's GPU at 8 frames.

**Why this goes first**
It is the only thing that makes the playground's most intuitive causal question — *does the answer
depend on audio?* — measurable, and it is the longest-lead item on the only hard deadline. It **carries
W9 and enriches W11** (see the dependency correction in §5 R1 and Appendix A: W11's rubric is
satisfiable without F1, so F1 makes W11 richer rather than gating it).

**Design note — the two cells are not redundant, and both carry teacher forcing**

| Cell | Question it asks | Character |
|---|---|---|
| Free generation (slow cell, W9) | *What would it say without this pathway?* | Demo. Visceral. String diff |
| Teacher forcing (**W9 slow cell, fixed params** + **W10 playground, student-driven**) | *How much less does it believe what it said?* | Measurement. Graded, deterministic, fast |

String diff is effectively binary (you can't see a *small* effect) and depends on the sampled
continuation; Δ log-lik is continuous and deterministic. **Keep both.** Note the scope this makes
explicit: **the W9 slow cell gains a fixed-parameter teacher-forced sub-pass** over the baseline
caption, so F2's per-token strip has data to render there — without it, F2's "works in both cells"
criterion is unbuildable (per-token Δ cannot exist without a teacher-forced pass).

### F3 — Null battery

**Problem**
The lens emits tokens no matter what you feed it. Until a student sees what it emits when there's nothing
to read, they'll mistake plausible output for evidence.

> **What "the lens" is here (state this in W8).** In this codebase the logit lens is
> `model.thinker.lm_head` applied to the **raw hidden states** — **no final RMSNorm**, no softmax
> (`src/logitlens_experiment.py:66,88-94`). That is a nonstandard *pre-norm* variant relative to the
> textbook logit lens and to Belrose et al. It has a direct consequence for C2 below, and it is worth
> one honest sentence in the W8 opener.

**C1 — Mismatched audio (the practical one).**
Swap in an unrelated soundtrack; if lens output at audio positions is unchanged → the lens is reading
**priors, not audio.** **Implementation (verified — this is *not* a file remux):** the processor
consumes audio as an explicit numpy array from `process_mm_info`, so `swap_audio(video_src, audio_src)`
extracts clip B's 16 kHz array (a second `process_mm_info` call, or ~10-15 decode-only PyAV lines — PyAV
is already a hard dependency), **trims/pads it to `len(audios[0])` samples**, and substitutes it before
the processor call, keeping **`use_audio_in_video=True`** so the token layout is position-identical to
the real run. Duration-matching is load-bearing: audio-token count scales with audio duration (even the
sample clip's own tracks differ — video 10.0 s vs audio 9.915 s). **Do not** feed the replacement as a
separate `{'type':'audio'}` element (`use_audio_in_video=False`) — it changes the chat template and the
audio/video interleaving, so "same plot type, side by side" breaks. The two-clip interface
(`video` from A, `audio` from B) is exactly the signature of the correct implementation.

**C2 — Random-direction activations (the shock).**
Feed the lens random-**direction** activations. If plausible tokens come out → **plausibility is not
evidence.** **Corrected spec (v1 said "norm-matched"; that word is inert here — see Appendix D):** the
lens is a **bias-free linear map with no final norm**, and every downstream metric is argmax/rank-based,
so scaling a hidden vector by any positive constant cannot change a single output — a "norm-matched"
Gaussian is mathematically identical to *any* isotropic Gaussian. The only real free parameter is the
**direction distribution**. So C2 is *isotropic random directions*, and its outcome is informative
**either way** (goal 1.1 #3): plausible tokens out → the shock lands; junk out → the lens needs an
*on-manifold* direction, which is itself a result and sets up C3.

**C3 — Shuffled real activations (the sharper control, disclosed alongside C2).**
Feed the lens **real** activations shuffled in from a different position / different clip. This
guarantees plausible output *by construction*, so it is the better **control** — but for that same
reason it carries **no shock** (unsurprising output). It asks a distinct question: *does the **content**
of this position matter, or does any audio activation decode to the same thing?* Nearly free against
this codebase: the hooks already store full `[batch, seq, d_model]` hidden states per layer on CPU, so
cross-position shuffling is a row permutation over tensors already in `logit_lens_storage`
(cross-clip needs one extra forward pass). Both C2 and C3 write the **existing CSV schema**, so F3's
"same plot type, comparable side by side" is automatic.

> **The ladder is curriculum, not a hidden knob.** v1 made "C2 produces plausible-looking output" an
> acceptance criterion backed by a *private* fallback chain iterated until the demo lands — which is
> experimenter degrees of freedom, the very habit W10 exists to teach against. Instead, **show students
> the whole ladder**: "we tried random directions; here is what came out and why; then we shuffled in
> real activations; here is what that tells us the first control couldn't." This is the same
> junk-set-as-discussion-material philosophy F4 already endorses, and it makes C1/C2/C3 three *distinct*
> lessons (priors-not-audio / plausibility-not-evidence / content-vs-any-activation), satisfying the
> non-redundancy test §2.2 applies to the cut silence/white-noise controls.

**Acceptance criteria**

- [ ] C1: `swap_audio(video_src, audio_src)` helper on the `use_audio_in_video=True` path,
  duration-matched; **rules out the separate-audio-element shortcut**.
- [ ] C2: function that generates isotropic random-direction activations at a given layer and runs the
  lens on them (no "norm-matching" in the spec).
- [ ] C3: helper that shuffles real activations across position / clip and runs the lens on them.
- [ ] All three emit the **same plot type** as the real run.
- [ ] The C2→C3 progression is presented to students as a **disclosed control-design ladder**, not a
  pass/fail on whether C2 "looks plausible."

### F4 — Metric ladder

**Problem**
The current metric (distinct token count per layer) isn't merely imprecise — **its sign may be wrong.**

In Qwen2VL-2B, only 6.5% of visual tokens map to the correct object class at the best layer (25.1 of 29;
Neo et al. 2025, verified). If audio behaves similarly, distinct count is **dominated by the 93.5% that
isn't signal.** Worse: real signal is *convergent* — on a piano clip, the positions carrying signal all
decode to `piano`, which **lowers** diversity. So an information-carrying pathway, when cut, can *raise*
diversity (positions scatter off the shared target) and a noise-injecting one can *lower* it — the
metric's interpretation can invert. The README currently reads a knockout-induced diversity drop as
"that pathway was carrying information" (`README.md:160-164`). **The signal you care about can push the
metric opposite to the metric's own stated interpretation.**

**Solution: a 3-rung ladder**

| Rung | Metric | Throws away | Status |
|---|---|---|---|
| 1 | distinct count | identity + probability mass | exists |
| 2 | **junk fraction** | — (cheap identity signal) | **build** |
| 3 | **target token rank** | — (needs ground truth) | **build** |

**junk fraction** — proportion of top-1 tokens that are punctuation / whitespace / high-frequency
function words / subword garbage. Pure **string classification over the CSVs that already exist** (zero
model-code change; the playground `_diversity` reader grows a junk column in a few lines; works offline
against the F5 precompute). Crossed with diversity, it separates the confound:

| | junk flat or ↓ | junk ↑ |
|---|---|---|
| **diversity ↓** | convergence (pathway may have been a distractor) | **degenerate collapse** (it just broke) |
| **diversity ↑** | representation differentiated | noise injection |

> **The 2×2 is a hypothesis generator, not a verdict.** Its "convergence = distractor" cell is **not
> decidable from diversity + junk alone** — convergence onto a generic *non-junk prior* word (e.g.
> `sound`) lands in the same cell as convergence onto the true target and means the opposite thing. Only
> **rung 3 (target rank)** arbitrates it. Frame it that way in class.

**target token rank** — you know what the sample clip is, so ground truth is free. **Cheaper than it
looks:** the full ~152k-dim logits are already materialized per position × layer *before* being
argmax'd and discarded (`src/logitlens_experiment.py:88-94`), so rank is **one added line**
(`(logits > logits[..., target_id]).sum()`) and a new CSV column — no new hooks, no logits storage
(that would be multi-GB). Two subtleties: (a) **"target token" is a BPE *set***, not a token — `piano`
/ ` piano` / `Piano` and multi-token splits have distinct ids, so compute **min-rank over a declared id
set** and surface that choice in the parameters cell (the same judgment-call lesson as the junk set);
(b) with `|V| ≈ 152k` the `|V|/2 ≈ 76k` null line needs a **log-scale y-axis** or rank-5-vs-76k is
invisible. **It has a principled null: ~|V|/2 under chance.**

**Acceptance criteria**

- [ ] Junk set defined and **documented** (it's a judgment call — see note).
- [ ] diversity × junk 2×2 plot or table.
- [ ] Per-layer target rank plot **with the |V|/2 null line drawn on a log axis**.
- [ ] Sample clip's target token **set** declared in the parameters cell; rank computed as min-rank over
  the set.
- [ ] **README revision (new deliverable).** F4 ships corrective metrics; the guidebook students read in
  W7-W9 must not keep teaching the reading W10 debunks. Revise `README.md:89` ("committed"/"deciding"),
  `:134` (audio→video "collapses → actively shaping"), `:160-164` (negative Δ = "carrying information"),
  **and suggested experiments 2 & 5** (`:171-173`, `:178-180`) — *either* soften to hypothesis language
  with forward-pointers to W10, *or* deliberately keep them as the artifact students critique in W10
  and say so in the instructor notes. Do not leave it an accidental contradiction. *(The notebook's
  playground copy is already interpretively neutral — it needs at most a one-line pointer, not a
  rewrite.)*
- [ ] **F5 coupling:** the precomputed CSVs must **bake rank + junk columns for the declared targets** —
  CSVs holding only top-1 strings cannot yield ranks retroactively.

**Design note — the arbitrariness of the junk set is a feature, not a bug**
What counts as junk is a judgment. Don't hide it — **use it as discussion material.** "Why this metric,
and what would be better?" is W10's question, and the junk set is a live instance of it.

## Appendix B — W11 assignment and rubric

**Announce the rubric in W9**, not W11 — incentives should shape two weeks of exploration, not land as a
surprise on the capstone.

**Two tracks**

- **Honest track** — find a real cross-modal dependency.
- **Adversarial track** — produce a large Δ that means nothing, and **prove it means nothing.** (Runs on
  the existing diversity metric; needs no F1.)

The adversarial track turns gaming into learning. A student who makes diversity scream while showing the
model merely broke has understood W10 completely.

**Grading — the (effect, control) pair**

| What the student submits | Marks |
|---|---|
| Large Δ + control showing it isn't collapse | Full |
| Large Δ + control revealing it's an artifact | **Full** (finding the artifact is a finding) |
| Null + control showing the method could have detected an effect | **Full** |
| Large Δ + no control | **Zero** |

**An effect without a control scores zero regardless of size.** That size is uninterpretable *is* the
thesis of the course. Grading on effect size would contradict W9 ("a null is a result") at the
gradebook, and would train the file-drawer habit.

**Guard against ritual controls (no new rubric rows).** Bolting on *any* control to dodge the zero must
not earn full marks. Grade the **control itself** with success criterion 1: the student must answer "if
my control passes, what is true; if it fails, what is true?" *for their own control* — a control whose
honest answer is "nothing either way" earns little. This turns the presence/absence rubric into a
quality rubric for free.

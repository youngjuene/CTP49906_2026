## 3. 빌드 명세

### F1 — 티처 포싱(`answer` 토큰 타입) 🔴 최우선

**문제**
플레이그라운드는 순전파(forward pass)를 한 번만 돌리므로 `generated` 토큰이 존재하지 않는다.
그런데 `generated → audio`는 학생이 가장 먼저 시도하는 것 — 가장 직관적인 규칙이다. 아무 일도
일어나지 않는다(노트북과 README가 이미 이를 *경고*해야 한다). 흥미를 잃는다. F1은 그 문서화된
막다른 길을, 채점 가능하고 연속적이며 결정론적인 측정으로 바꾼다.

**실현 가능성(검증됨).** 어려운 부분은 이미 존재하므로, v1의 "가장 위험한 코드"라는 표현은
난이도를 과장한다 — 8월 31일 일정 산정에 알아 둘 값어치가 있다. `BlockAttentionHook`의 **프리필
분기(prefill branch)**는 `q_len == k_len == original_input_len`일 때마다 발동하므로
(`src/attention_knockout_experiment.py:117-144`), 확장된 길이로 `[prompt, C]`를 순전파하면
플레이그라운드가 이미 매일 구동하는 바로 그 마스크 코드 경로를 그대로 탄다. 덧붙이는 것은
`input_ids` + `attention_mask`에 대한 `torch.cat`뿐이다 — 오디오/비디오 특징 텐서는 손대지 않는다
(플레이스홀더 산포는 토큰 id로 매칭되고, 뒤따르는 텍스트는 생성 때와 같은 티처 포싱된 RoPE
위치를 받는다). 위치별 `log P(C_t | C_<t)`는 순전파 로짓에 대한 `log_softmax` 한 번을 한 칸
민 것이다.

**해법**

1. **플레이그라운드 실행 셀에서**(이미 로드된 **eager** `attention_model` 사용) 기준 캡션 `C`를
   **그리디 디코딩**(`do_sample=False`)으로 생성하되, `(clip hash, prompt, nframes)`를 키로
   **캐시**해 바뀌지 않은 제출은 재생성을 건너뛴다. **생 생성 id**를 슬라이스한다:
   `C_ids = gen[:, input_len:]`. **디코딩된 문자열을 다시 토크나이즈하지 마라** —
   `batch_decode(..., skip_special_tokens=True)`는 오디오/비디오 플레이스홀더를 떨어뜨리고
   왕복(round-trip)하지 않아 특징 산포가 깨진다. *(v1은 "느린 셀에서 C를 한 번 생성"이라고 했는데,
   그 캡션은 파라미터 셀의 클립/프롬프트에 묶여 있어 학생이 업로드한 무엇이든 그 위에 티처
   포싱된다 — 자기 신뢰 Δ로 잘못 라벨링된 불일치 오디오 측정이며, W11의 학생 클립을 깨뜨린다.)*
2. `C_ids`를 덧붙이고 `attention_mask = cat([mask, ones])`. `[prompt, C]`에 대해 순전파.
3. `C`의 위치를 **`answer`**로 태그한다 — **위치 기반으로**(`prompt_types + ["answer"] *
   len(C_ids)`), 토큰 id 매퍼를 다시 돌리지 **말 것**(그러면 `query_text`로 태깅된다). `"answer": 5`를
   `TOKEN_TYPE_MAP`에 추가한다.
4. `original_input_len =` **전체 확장 길이**를 넘겨 프리필 분기가 발동하도록 한다
   (`q_len == k_len == extended_len`). 프롬프트만의 길이로는 어느 훅 분기도 발동하지 않아 녹아웃이
   조용히 무효화된다.
5. 녹아웃 `answer → audio`를 레이어 `[a, b)`에 적용. **녹아웃 패스와 기준 패스**를 각각 돌린다(같은
   eager 모델, 같은 인코딩 입력, 둘 다 그리디).
6. 각 `C` 위치에서 로짓을 읽어 녹아웃과 기준에 대해 `Σ_t log P(C_t | C_<t, prompt)`를 계산한다.
   지표: **Δ log-우도(log-likelihood), `Δ = knockout − baseline`** — 모델이 자기가 한 말을 얼마나
   덜 믿게 되는가. *(부호 규약은 기존 "Δ diversity (knockout − baseline)" 패널에 맞춰 고정.)*

**가드레일(조용히가 아니라 시끄럽게 실패).** `block_attention`의 `KeyError` 처리
(`src/attention_knockout_experiment.py:184-196`)를 **출력하고 훅 0개로 yield**하던 것에서
`raise`로 바꾼다. 지금 상태로는 오타나 미지의 토큰 타입이 **녹아웃으로 라벨링된 기준**을 돌린다 —
이 과목이 학생들에게 불신하라고 가르치는 바로 그 그럴듯하지만-틀린 산물이다. 플레이그라운드의
`except Exception` 처리가 이미 raise를 노트북 내 콜아웃으로 드러낸다. `block_attention`에 들어가는
티처 포싱 타입은 반드시 어텐션 매퍼(위치 기반 `answer`)에서 와야 하며, logit-lens id 매퍼에서
오면 안 된다.

**인수 기준**

- [ ] `answer`가 `TOKEN_TYPE_MAP`에 나타나고 `token counts:`에 출력된다; 드롭다운에 `answer`가 보인다.
- [ ] **새 소스를 위한 학생 대면 표면 갱신**(F4의 README 개정 산출물과 병렬). 티처 포싱은 **자체**
  플레이그라운드 섹션이다(소스는 `answer`로 고정, 학생이 타깃과 레이어 대역을 고름) — *다양성
  스코어보드* 드롭다운(`CTP49906_avllm_molab.py:607,711`)에 새 옵션으로 넣지 **않는다**. 거기서는
  `answer`가 `generated`와 똑같이 무효이기 때문이다(그 스코어보드는 answer 토큰이 없는 프롬프트-only
  순전파를 돈다). README 토큰 타입 표에 `answer`를 추가하고 "Why `generated` does nothing" 콜아웃
  (`README.md:113-141`)을, `answer` 소스가 의미를 갖게 되는 지점이 티처 포싱임을 가리키도록 개정한다.
- [ ] `answer → audio`가 오디오 의존적 캡션을 가진 클립에서 **정해진 음의 마진 아래**의 총합 Δ를
  낸다(단순히 "0이 아님"이 아님 — 키가 하나라도 마스킹되면 부동소수 잡음이 "0 아님"을 통과시킴).
- [ ] **반증 가능하게 만든 안전성 검사:** 커밋된 **무음 트랙** 에셋(오디오 스트림이 *존재*하되 전부
  0 — 그래서 막을 실제 오디오 토큰이 존재)에서, 같은 프레임·프롬프트·레이어로
  `|Δ_silent| / 토큰 ≪ |Δ_audio-dependent| / 토큰`을 정해진 허용 오차 내에서 만족한다. *(오디오
  트랙이 **없는** 클립은 오디오 토큰을 0개 만들어 `answer → audio`가 아무것도 막지 못하므로 Δ = 0이
  공허하게 성립한다 — 티처 포싱이 완전히 고장 나도 통과한다. 트랙 없는 클립은 대신 기존의 오디오
  토큰 없음 경고를 띄워야 하며, 이 검사의 대체물이 아니다.)*
- [ ] **결정론적:** `C`를 그리디로 디코딩하며 `do_sample=False`를 **명시적으로** 고정 — 배포된
  `generation_config.json`은 오늘날 그리디로 귀결되는 빈 스텁이므로, 이는 조용한 상류 변경(주간
  8 GB 재다운로드 R5가 곧장 수업으로 배달할 수 있음)에 대한 명세 강화이지 실제 버그 수정이 아니다.
  검사: 같은 제출을 두 번 돌려 **토큰별 Δ 벡터가 비트 단위로 동일**함을 단언한다(같은 모델 인스턴스,
  재인코딩 없음).
- [ ] **제출당 예산**("순전파 한 번"이 아님 — Δ는 녹아웃 + 기준 = 둘이 필요): 캐시가 데워진 제출이
  molab GPU에서 8프레임 기준 **≤ 순전파 2회 / ≤ N초** 안에 완료된다.

**왜 이것이 먼저인가**
플레이그라운드의 가장 직관적인 인과 질문 — *답변이 오디오에 의존하는가?* — 을 측정 가능하게 만드는
유일한 것이자, 유일한 엄수 마감에 걸린 가장 긴 리드타임 항목이다. **W9를 떠받치고 W11을 풍부하게
한다**(§5 R1과 부록 A의 의존성 정정 참고: W11의 루브릭은 F1 없이도 충족 가능하므로, F1은 W11을
가로막는 게 아니라 더 풍부하게 만든다).

**설계 노트 — 두 셀은 중복이 아니며, 둘 다 티처 포싱을 담는다**

| 셀 | 던지는 질문 | 성격 |
|---|---|---|
| 자유 생성(느린 셀, W9) | *이 경로 없이 무엇을 말할까?* | 데모. 체감적. 문자열 diff |
| 티처 포싱(**W9 느린 셀, 고정 파라미터** + **W10 플레이그라운드, 학생 주도**) | *자기가 한 말을 얼마나 덜 믿는가?* | 측정. 채점 가능, 결정론적, 빠름 |

문자열 diff는 사실상 이진적이고(*작은* 효과는 볼 수 없음) 샘플링된 이어쓰기에 의존한다; Δ log-lik은
연속적이고 결정론적이다. **둘 다 유지하라.** 이것이 명시화하는 범위: **W9 느린 셀은 기준 캡션에
대한 고정 파라미터 티처 포싱 서브패스를 갖게 된다**. 그래야 F2의 토큰별 스트립이 거기서 렌더링할
데이터를 가진다 — 그것 없이는 F2의 "두 셀 모두에서 작동" 기준을 만들 수 없다(토큰별 Δ는 티처 포싱
패스 없이 존재할 수 없음).

### F2 — 토큰별 Δ log-lik 시각화

**문제**
총합 Δ는 스칼라다. 캡션의 *어느 단어*가 오디오에 의존했는지 볼 수 없다.

**해법**
캡션 텍스트를 토큰별 Δ log-lik으로 색칠한다. `a person is playing the piano`에서: `piano`는 3 nat
떨어지고 나머지는 0.01 → **`piano`는 뜨겁게 타오르고 나머지는 회색으로 남는다.** **새 의존성 없이**
가능하다: 노트북은 이미 matplotlib 그림을 렌더링하고 느린 셀과 플레이그라운드 양쪽에서
`mo.md`/`mo.vstack`으로 리치 출력을 구성하므로, 스트립은 인라인 HTML span 나열이다(배경은 matplotlib
`RdBu` + `TwoSlopeNorm(vcenter=0)`, **음수** 쪽이 뜨거운 색이 되도록 방향을 맞춤; 토큰별 nat 값은
`title` 호버 속성으로).

**인수 기준**

- [ ] 캡션이 토큰별 색상 스케일로 렌더링된다, **발산형(diverging), 0 중심**(Δ는 잡음으로 양수가 될
  수 있음).
- [ ] **부호 규약 고정:** `Δ = knockout − baseline`, 따라서 **음수 = 녹아웃 후 덜 믿음 = 뜨거운
  색**이며, 기존 "Δ diversity (knockout − baseline)" 패널 및 F1 지표와 일치한다. 두 패널이 절대
  어긋나지 않도록 셀 텍스트 *와* README 양쪽에 명시한다.
- [ ] 실제 nat 값을 토큰마다 호버 또는 라벨로 노출.
- [ ] **10-15 토큰 캡션에서 가독, 서브워드 처리 명세:** 표시는 **단어 단위로 이은** span, 호버는
  **토큰 단위** 값 유지. *(Qwen2.5-Omni 토크나이저에서 ` piano`는 단일 토큰이라 시연 단어는 잘
  렌더링되지만, ` saxophone` → `sax`+`ophone`, ` ukulele` → 3조각으로 쪼개지므로, 가공 없는 토큰별
  스트립은 단어가 아니라 조각 경계를 보여 준다.)*
- [ ] W9 느린 셀(고정 파라미터)과 플레이그라운드(학생 주도) 양쪽에서 작동.

**우선순위 노트**
이것이 **학생들이 유일하게 기억할 이미지**다. "정말 듣는가?"에 대한 한 장의 답이자, W9의 "널도
결과다"가 시각적으로 착지하는 지점이다. 시간을 들여라.

**의존:** F1.

### F3 — 널 배터리

**문제**
렌즈는 무엇을 넣든 토큰을 뱉는다. 읽을 게 아무것도 없을 때 렌즈가 무엇을 뱉는지 학생이 볼 때까지,
그들은 그럴듯한 출력을 증거로 오인한다.

> **여기서 "렌즈"가 무엇인가(W8에서 밝혀라).** 이 코드베이스에서 logit lens는 **생 은닉 상태(raw
> hidden states)**에 적용된 `model.thinker.lm_head`다 — **최종 RMSNorm 없음**, 소프트맥스 없음
> (`src/logitlens_experiment.py:66,88-94`). 이는 교과서적 logit lens 및 Belrose et al.에 대해
> 비표준적인 *프리-노름(pre-norm)* 변형이다. 아래 C2에 직접적인 결과를 미치며, W8 오프닝에서 솔직한
> 한 문장의 값어치가 있다.

**C1 — 불일치 오디오(실용적인 것).**
무관한 사운드트랙으로 교체한다; 오디오 위치의 렌즈 출력이 그대로면 → 렌즈는 **오디오가 아니라
사전 지식(prior)을 읽고 있다.** **구현(검증됨 — 파일 리먹스가 *아님*):** 프로세서는 오디오를
`process_mm_info`에서 나온 명시적 numpy 배열로 소비하므로, `swap_audio(video_src, audio_src)`는
클립 B의 16 kHz 배열을 추출해(두 번째 `process_mm_info` 호출, 혹은 디코드 전용 ~10-15줄 PyAV — PyAV는
이미 필수 의존성) **`len(audios[0])` 샘플로 자르거나 채우고**, 프로세서 호출 전에 갈아끼우되
**`use_audio_in_video=True`**를 유지해 토큰 배치가 실제 실행과 위치까지 동일하게 한다. 길이 맞춤이
관건이다: 오디오 토큰 수는 오디오 길이에 비례한다(샘플 클립조차 자체 트랙이 다름 — 비디오 10.0초 vs
오디오 9.915초). 교체본을 별도의 `{'type':'audio'}` 요소(`use_audio_in_video=False`)로 넣지
**말 것** — 채팅 템플릿과 오디오/비디오 인터리빙을 바꿔 "같은 그래프 유형, 나란히"가 깨진다. 두 클립
인터페이스(`video`는 A에서, `audio`는 B에서)가 바로 올바른 구현의 시그니처다.

**C2 — 무작위 방향 활성(충격).**
렌즈에 무작위 **방향** 활성을 먹인다. 그럴듯한 토큰이 나오면 → **그럴듯함은 증거가 아니다.**
**수정된 명세(v1은 "노름 정합"이라 했으나 여기선 그 단어가 무효 — 부록 D 참고):** 렌즈는 **최종
노름이 없는 편향 없는 선형 사상**이고 하류 지표가 전부 argmax/rank 기반이므로, 은닉 벡터를 어떤 양의
상수로 스케일해도 출력이 단 하나도 바뀌지 않는다 — "노름 정합" 가우시안은 *임의의* 등방
가우시안과 수학적으로 동일하다. 유일한 실제 자유 파라미터는 **방향 분포**다. 그래서 C2는 *등방
무작위 방향*이고, 그 결과는 **어느 쪽이든** 유익하다(목표 1.1 #3): 그럴듯한 토큰이 나오면 → 충격이
착지한다; 쓰레기가 나오면 → 렌즈는 *매니폴드 위(on-manifold)* 방향을 필요로 한다는 것이 그 자체로
결과이고 C3의 무대를 깐다.

**C3 — 실제 활성 셔플(더 날카로운 대조군, C2와 함께 공개).**
다른 위치 / 다른 클립에서 가져온 **실제** 활성을 셔플해 렌즈에 먹인다. 이는 그럴듯한 출력을 *구성상*
보장하므로 더 나은 **대조군**이다 — 그러나 바로 그 이유로 **충격은 없다**(놀랍지 않은 출력). 별개의
질문을 던진다: *이 위치의 **내용**이 중요한가, 아니면 아무 오디오 활성이나 같은 것으로 디코딩되는가?*
이 코드베이스에서는 거의 공짜다: 훅이 이미 레이어별 전체 `[batch, seq, d_model]` 은닉 상태를 CPU에
저장하므로, 위치 간 셔플은 이미 `logit_lens_storage`에 있는 텐서에 대한 행 치환이다(클립 간은 순전파
한 번 추가). C2와 C3 모두 **기존 CSV 스키마**로 쓰므로 F3의 "같은 그래프 유형, 나란히 비교 가능"이
자동으로 성립한다.

> **사다리는 커리큘럼이지 숨겨진 노브가 아니다.** v1은 "C2가 그럴듯한 출력을 낸다"를, 데모가 착지할
> 때까지 반복하는 *비공개* 폴백 체인으로 뒷받침되는 인수 기준으로 삼았다 — 이는 실험자 자유도
> (experimenter degrees of freedom), 바로 W10이 맞서 가르치는 습관이다. 대신 **사다리 전체를
> 학생에게 보여라**: "무작위 방향을 시도했고, 이런 게 왜 나왔는지; 그다음 실제 활성을 셔플했고, 첫
> 대조군이 못 한 것을 이게 무엇을 말해 주는지." 이는 F4가 이미 지지하는 junk-set-을-토론
> 자료로-삼기 철학과 같으며, C1/C2/C3를 세 개의 *별개* 교훈(사전 지식이지 오디오가 아님 / 그럴듯함은
> 증거가 아님 / 내용 대 아무 활성)으로 만들어, §2.2가 잘라낸 무음/백색잡음 대조군에 적용하는
> 비중복성 검사를 만족한다.

**인수 기준**

- [ ] C1: `use_audio_in_video=True` 경로 위의 `swap_audio(video_src, audio_src)` 헬퍼, 길이 맞춤;
  **별도 오디오 요소 지름길을 배제**.
- [ ] C2: 주어진 레이어에서 등방 무작위 방향 활성을 생성해 렌즈에 태우는 함수(명세에 "노름 정합"
  없음).
- [ ] C3: 위치 / 클립을 가로질러 실제 활성을 셔플해 렌즈에 태우는 헬퍼.
- [ ] 셋 다 실제 실행과 **같은 그래프 유형**을 낸다.
- [ ] C2→C3 진행을 학생에게 **공개된 대조군 설계 사다리**로 제시하며, C2가 "그럴듯해 보이는지"에
  대한 합격/불합격이 아니다.

### F4 — 지표 사다리

**문제**
현재 지표(레이어별 서로 다른 토큰 수)는 단지 부정확한 정도가 아니라 — **부호가 틀렸을 수 있다.**

Qwen2VL-2B에서는 시각 토큰의 6.5%만이 최적 레이어(29개 중 25.1번째)에서 올바른 객체 클래스로
사상된다(Neo et al. 2025, 검증됨). 오디오가 비슷하게 행동한다면, 서로 다른 토큰 수는 **신호가 아닌
93.5%에 지배된다.** 더 나쁜 것은: 실제 신호는 *수렴적*이다 — 피아노 클립에서 신호를 나르는 위치들은
모두 `piano`로 디코딩되어 다양성을 **낮춘다.** 그래서 정보를 나르는 경로는 잘렸을 때 다양성을 *올릴*
수 있고(위치들이 공유 타깃에서 흩어짐), 잡음을 주입하는 경로는 *낮출* 수 있다 — 지표의 해석이
뒤집힐 수 있다. README는 현재 녹아웃으로 인한 다양성 하락을 "그 경로가 정보를 나르고 있었다"로 읽는다
(`README.md:160-164`). **당신이 관심 있는 신호가 지표를 그 지표 자신의 서술과 반대로 밀 수 있다.**

**해법: 3단 사다리**

| 단 | 지표 | 버리는 것 | 상태 |
|---|---|---|---|
| 1 | 서로 다른 개수 | 토큰 정체(어느 토큰인지) + 확률 질량 | 존재 |
| 2 | **junk fraction** | — (값싼 토큰-정체 신호) | **빌드** |
| 3 | **target token rank** | — (정답이 필요) | **빌드** |

**junk fraction** — top-1 토큰 중 구두점 / 공백 / 고빈도 기능어 / 서브워드 쓰레기의 비율. **이미
존재하는 CSV에 대한 순수 문자열 분류**다(모델 코드 변경 0; 플레이그라운드 `_diversity` 리더가 몇 줄로
junk 열을 얻음; F5 사전 계산에 대해 오프라인으로 동작). 다양성과 교차하면 혼란 요인을 분리한다:

| | junk 평탄 또는 ↓ | junk ↑ |
|---|---|---|
| **다양성 ↓** | 수렴(경로가 방해물이었을 수도) | **퇴행적 붕괴**(그냥 고장 남) |
| **다양성 ↑** | 표현이 분화됨 | 잡음 주입 |

> **이 2×2는 판결이 아니라 가설 생성기다.** "수렴 = 방해물" 칸은 **다양성 + junk만으로는 결정
> 불가능**하다 — 일반적인 *비-junk 사전 지식* 단어(예: `sound`)로의 수렴은 진짜 타깃으로의 수렴과
> 같은 칸에 떨어지지만 반대를 의미한다. 오직 **3단(target rank)**만이 이를 판가름한다. 수업에서 그렇게
> 틀 지어라.

**target token rank** — 샘플 클립이 무엇인지 알므로 정답이 공짜다. **보기보다 값싸다:** 전체
~152k 차원 로짓이 argmax되어 버려지기 *전에* 위치 × 레이어별로 이미 산출되어(메모리에 실체화되어) 있으므로
(`src/logitlens_experiment.py:88-94`), rank는 **한 줄 추가**(`(logits > logits[..., target_id]).sum()`)와
새 CSV 열이다 — 새 훅 없음, 로짓 저장 없음(그건 수 GB가 됨). 두 가지 미묘함: (a) **"target
token"은 BPE *집합***이지 토큰이 아니다 — `piano` / ` piano` / `Piano`와 다중 토큰 분할이 서로 다른
id를 가지므로, **선언된 id 집합에 대한 최소 rank**를 계산하고 그 선택을 파라미터 셀에 드러낸다(junk
셋과 같은, 판단이 필요한 결정이라는 교훈); (b) `|V| ≈ 152k`이므로 `|V|/2 ≈ 76k` 널 선은 **로그 스케일 y축**이
필요하다. 안 그러면 rank-5 대 76k가 안 보인다. **원칙적 널을 갖는다: 우연 하에 ~|V|/2.**

**인수 기준**

- [ ] junk 셋 정의 및 **문서화**(주관적 판단이다 — 노트 참고).
- [ ] 다양성 × junk 2×2 그래프 또는 표.
- [ ] 레이어별 target rank 그래프 **에 |V|/2 널 선을 로그 축에 그림**.
- [ ] 샘플 클립의 target token **집합**을 파라미터 셀에 선언; rank는 집합에 대한 최소 rank로 계산.
- [ ] **README 개정(신규 산출물).** F4는 교정 지표를 내놓는다; W7-W9에 학생이 읽는 안내서가 W10이
  반증하는 해석을 계속 가르쳐서는 안 된다. `README.md:89`("committed"/"deciding"), `:134`(audio→video
  "collapses → actively shaping"), `:160-164`(음의 Δ = "carrying information"), **그리고 제안 실험 2 &
  5**(`:171-173`, `:178-180`)를 개정한다 — *혹은* W10으로의 전방 포인터와 함께 가설 어조로 누그러뜨리거나,
  *혹은* 의도적으로 W10에서 학생이 비판할 자료로 남기고 강사 노트에 그렇게 적는다. 우발적 모순으로
  방치하지 마라. *(노트북의 플레이그라운드 문구는 이미 해석적으로 중립적이다 — 재작성이 아니라 한 줄
  포인터면 족하다.)*
- [ ] **F5 결합:** 사전 계산된 CSV는 **선언된 타깃에 대한 rank + junk 열을 구워 넣어야** 한다 —
  top-1 문자열만 담은 CSV로는 rank를 소급해 낼 수 없다.

**설계 노트 — junk 셋의 자의성은 버그가 아니라 특징이다**
무엇이 junk인지는 판단이다. 숨기지 마라 — **토론 자료로 써라.** "왜 이 지표이고, 무엇이 더 나을까?"가
W10의 질문이며, junk 셋은 그것의 살아 있는 사례다.

### F5 — 사전 계산 결과(F5a / F5b로 분할)

**문제**
무료 GPU는 5주 중 최소 한 번은 고장 난다. 이는 보험이 아니라 요구사항이다. 노트북은 현재 어떤
그래프나 플레이그라운드 폼이 렌더링되기 전에 **CUDA를 강제로 단언한다**
(`CTP49906_avllm_molab.py:195`). 그래서 molab 장애는 수업을 통째로 죽인다.

**해법 — 이진 동결이 아니라 라이브 폴백을 갖춘 캐시**
샘플 클립에 대해 전 레이어 × (기준 + 카탈로그의 각 녹아웃 규칙)의 CSV와 그래프용 산물을 커밋한다.
고정 셀 데이터는 아주 작다(CSV당 ~1 MB 미만; 샘플 클립은 오디오 위치 ~250개).

**F5a(지금 착수 가능 — 8월 중순 목표).** **이미 존재하고 아무것에도 의존하지 않는** 그래프를 사전
계산한다: W7의 토큰 타입 맵 / `token counts:`, W8의 다양성/우세도 그래프(`:401-419`), W9의
기준-대-녹아웃 diff(`:434-492`) + 어텐션 히트맵(`:508-557`). 이들은 F1-F4에 의존하지 않는다. F5a를
일찍 내놓는 것이 실제로 가장 많은 수업 시간을 지키며, v1의 "F1이 밀리면 전부 밀린다" 서술을 자기
실현적으로 만들던 F1→F5 연쇄 위험을 제거한다.

**F5b(9월 30일).** 사전 계산을 F1/F2/F4 출력(토큰별 Δ log-lik JSON, rank/junk CSV 열)으로 확장하고
**캔드 모드 플레이그라운드**를 만든다.

**인수 기준**

- [ ] `USE_PRECOMPUTED`가 **`False`가 기본값**이며 **비상용(break-glass)**으로 문서화된다. *(v1의 문자
  그대로의 기준 "노트북에 `USE_PRECOMPUTED = True` 플래그"는 켜진 채 배포하는 것으로 읽히며 — 그러면
  W7-W10이 슬라이드쇼가 되고 학생의 첫 라이브 모델 상호작용이 W11로 밀린다.)*
- [ ] 사전 계산 모드에서 **W7-W10의 모든 *비대화형* 그래프가 GPU 없이 렌더링**되고, 각각 눈에 보이는
  **"replayed from cache"** 라벨을 단다. *(그럴듯한 출력과 라이브 증거를 구분하는 과목에 주제적으로도
  들어맞는다.)*
- [ ] **플레이그라운드는 라이브 유지** — 임의의 업로드/프롬프트/규칙에 대해 조용히 캐시를 내놓지 않고
  **시끄럽게 실패**한다; 선택적으로 `(clip, prompt, nframes, rules)`가 커밋된 그리드와 일치하면
  **캐시 히트**를 라벨과 함께 내놓고, 아니면 라이브로 돈다. 절대 조용히 대체하지 않는다.
- [ ] **생 피클이 아니라 산물:** 실행별 logit-lens CSV(기존 스키마 + rank/junk 열), 기준/녹아웃 캡션
  텍스트 + 토큰별 Δ log-lik JSON, 그리고 **요약된 어텐션 질량 행렬**. **생 캡처 어텐션 텐서는 절대
  커밋하지 말 것** — seq ≈ 1k인 eager 프리필 레이어 하나가 fp32로 ~64 MB라, 그것만으로 40 MB 선을
  넘는다.
- [ ] 데이터를 저장소에 커밋(어떤 파일이 ~40 MB를 넘으면 외부 호스팅 + 링크).

**노트 — 진짜 작업은 데이터 생성이 아니라 셀 게이팅이다.** 그래프 셀은 marimo DAG에서 모델 로딩 셀의
하류에 있고(플레이그라운드 폼은 레이어 수를 위해 `:606`에서 `attention_model`을 읽기까지 한다),
노트북은 앞머리에서 CUDA를 강제 단언한다. `USE_PRECOMPUTED`는 그래프 셀 안에서 분기하는 게 아니라
**모델 로드 셀을 단락(short-circuit)**시켜야 한다(또는 레이어 수를 스텁으로). F5 예산을 그에 맞춰
잡아라.

**부수 효과(중요).** F5a+F5b가 동작하면 **세션 중 GPU가 필요한 주는 W11뿐**이며, 학생의 molab
세션이 죽어도 W7-W10을 잃지 않는다.

**한계.** W11은 학생이 가져온 클립을 쓰므로 사전 계산할 수 없다. 예상된 바다 — 그 GPU 리스크는 F5가
아니라 R8이 다룬다.

## 부록 B — W11 과제와 루브릭

**루브릭을 W9에 공지하라**, W11이 아니라 — 인센티브는 탐구의 두 주를 형성해야지 캡스톤에 깜짝
등장해서는 안 된다.

**두 트랙**

- **정직 트랙** — 실제 교차 모달 의존성을 찾는다.
- **적대적 트랙** — 아무 의미 없는 큰 Δ를 만들고, **그것이 아무 의미 없음을 증명한다.** (기존 다양성
  지표로 돌아감; F1 불필요.)

적대적 트랙은 게이밍을 학습으로 바꾼다. 다양성을 비명 지르게 만들면서 모델이 그냥 고장 났음을 보이는
학생은 W10을 완전히 이해한 것이다.

**채점 — (효과, 대조군) 쌍**

| 학생이 제출하는 것 | 점수 |
|---|---|
| 큰 Δ + 붕괴가 아님을 보이는 대조군 | 만점 |
| 큰 Δ + 그것이 인공물임을 드러내는 대조군 | **만점**(인공물을 찾은 것도 발견) |
| 널 + 방법이 효과를 탐지할 수 있었음을 보이는 대조군 | **만점** |
| 큰 Δ + 대조군 없음 | **0점** |

**대조군 없는 효과는 크기와 무관하게 0점이다.** 그 크기가 해석 불가능하다는 것 *자체가* 이 과목의
논지다. 효과 크기로 채점하면 W9("널도 결과다")를 채점표에서 반박하게 되고, 파일 서랍(file-drawer)
습관을 훈련시킨다.

**의례적 대조군 방지(새 루브릭 행 없음).** 0점을 피하려 *아무* 대조군이나 갖다 붙이는 것이 만점을
받아서는 안 된다. **대조군 자체**를 성공 기준 1로 채점하라: 학생은 자신의 대조군에 대해 "내 대조군이
통과하면 무엇이 참이고, 실패하면 무엇이 참인가?"에 답해야 한다 — 정직한 답이 "어느 쪽이든 아무것도"인
대조군은 낮은 점수를 받는다. 이렇게 하면 있음/없음 루브릭이 공짜로 품질 루브릭이 된다.
